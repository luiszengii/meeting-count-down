## Design Context

### Users
主要用户是希望给会议提醒增加一点趣味性和仪式感的 macOS 用户。他们的核心场景不是管理复杂排期，而是在进入会议前获得一个更有存在感、更好玩的提醒体验。当前第一批用户以飞书用户为主，接入路径固定为 `Feishu CalDAV -> macOS Calendar -> 本地菜单栏 app`；未来产品语义可以保留“会议来自多个平台”的扩展空间，但当前设计不应暗示已经支持多源接入。

### Brand Personality
产品人格用三个词概括：`易用`、`俏皮`、`愉悦`。它不是严肃的企业后台，也不是冷冰冰的效率工具，而是一个把“准备进入会议”这件事变得更有趣的小型陪伴式 utility。界面应传递一点轻松感和趣味感，允许用户感受到“这个提醒是有性格的”，但不能幼稚、吵闹或妨碍效率。

### Aesthetic Direction
视觉方向以最新 macOS 的 `liquid glass` 语言为主，优先保留原生桌面应用的质感、层次和可信度，再在主题色与细节节奏上做出一点区别于系统默认控制面的个性。整体应像一款真正属于 macOS 生态的现代菜单栏应用，而不是网页移植出来的面板。明确避免：

- 枯燥、过度工具化的内部后台观感
- 一眼可辨的 AI 模板味
- 只靠深色毛玻璃堆砌出来的空洞“高级感”

### Design Principles
- 原生优先：优先做成像 macOS 自己会提供的菜单栏体验，再考虑品牌化修饰。
- 趣味点到即止：提醒和倒计时可以带一点玩味和记忆点，但不能喧宾夺主或牺牲可读性。
- 状态先于装饰：用户最先看到的必须是“下一场会议是什么、什么时候开始、提醒是否会生效”。
- 避免工具腔：减少管理面板、设置堆砌、后台表单式观感，让界面更像一个有情绪的小助手。
- 主题色克制但有记忆点：允许在液态玻璃基底上使用更有个性的主题色搭配，但不要走成通用 AI 渐变或过饱和炫技风格。
- 当前阶段不过度为可访问性设硬指标，但动效和颜色仍应保持克制，不做高频闪烁或负担过重的视觉噪音。
