# 2026-03-30 强化学习导向的 Swift 注释密度要求

- 状态：Accepted
- 日期：2026-03-30
- 关联日志：[2026-03-30 开发日志](../dev-logs/2026-03-30.md)

## 背景

项目已经进入 Swift 代码实现阶段，但当前代码主要只有类型级注释，函数级注释明显不足。由于项目维护者当前对 Swift 语法、Apple 框架和 macOS 生命周期都不熟悉，如果注释只停留在文件和类型级别，阅读成本仍然很高，无法达到“边开发边学习 Swift 生态”的目标。

## 决策

从现在开始，项目的 Swift 注释标准提升为“学习导向的详细中文注释”：

1. 函数级注释默认必写，尤其是带分支、状态变更、异步边界、格式化逻辑、测试构造逻辑、Apple 框架交互的函数。
2. 关键计算属性和关键测试用例也应写注释。
3. 注释密度目标约为每 `8` 到 `10` 行有效代码至少有 `1` 行高信息量注释。
4. 注释默认面向“不熟悉 Swift 的读者”，需要解释语义和上下文，而不只是业务名称。

## 备选方案

### 方案 A：保持现有类型级注释

成本最低，但仍然不足以帮助不熟悉 Swift 的读者理解代码。

### 方案 B：只在复杂文件中提高注释密度

比当前更好，但标准不统一，容易让不同 session 的实现质量出现漂移。

### 方案 C：逐行注释所有代码

信息量表面上更高，但噪音也更大，容易让注释变成机械翻译。

## 影响

- 现有和未来 Swift 文件都需要补更多函数级注释。
- 开发速度会略慢，但可读性、可维护性和学习价值会明显提高。
- Review 标准会从“有没有注释”升级成“注释是否足以帮助不熟悉 Swift 的读者理解”。

## 后续动作

1. 更新根级 `AGENTS.md` 和模板，固化这一要求。
2. 回补当前已存在 Swift 文件的函数级注释。
3. 后续新增模块时，在目录级 `AGENTS.md` 中也复述该目录的注释要求。
