## 当天开发概括

修正设置窗口顶部 tab 导航的选中态可读性：让浅色选中胶囊里的文字稳定落到深灰，并补上文字与胶囊同步切换的轻量动画，避免当前选中项在截图场景里发白发虚。

## 详细内容

### 1. 完成事项

- 更新 [MeetingCountdownApp/AppShell/GlassUI.swift](../../MeetingCountdownApp/AppShell/GlassUI.swift)，调整 `GlassSegmentedTabs` 的选中态文字颜色，不再让选中标签继续使用接近白色的前景。
- 为同一处 tab 切换补上专用动效常量，让选中底板、文字颜色和轻微缩放一起过渡，减少“底板动了、字色硬切”的割裂感。

### 2. 关键决策

- 这组设置页 tab 的选中态对比度优先服从“浅色胶囊上的可读性”，因此选中文字显式使用深灰中性色，而不是继续跟随当前玻璃背景走近白色前景。
- 动效继续保持克制，只补短时的颜色与缩放过渡，不引入弹跳或高存在感反馈，避免破坏当前 macOS 风格的稳定感。

### 3. 遇到的问题

- 当前 tab 导航选中项虽然有浅色胶囊，但文字也偏亮，导致“概览 / 日历 / 提醒 / 音频 / 高级”这类短标签在截图和高亮背景下对比度不足。
