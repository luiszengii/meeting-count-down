- 相关问题已经记录到 [xcodebuild 首次运行时插件加载失败](../pitfalls/xcodebuild-first-launch-plugin-failure.md)。

### 7. 强化 Swift 注释密度规则

- 当前代码已经具备文件级和类型级注释，但函数级注释不足。
- 现在把规则升级为“学习导向的详细中文注释”：函数级注释默认必写，目标密度约为每 `8` 到 `10` 行有效代码至少有 `1` 行高信息量注释。
- 这项规则升级已经记录到新的 ADR，后续应回补当前 Swift 文件的注释缺口。

相关文档：
- [强化学习导向的 Swift 注释密度要求](../adrs/2026-03-30-learning-oriented-swift-comment-density.md)

### 8. 第二轮注释回补与局部规则加固

- 继续回补现有 Swift 文件的字段级、计算属性级和测试辅助对象注释，避免只停留在类型级注释。
- 为 `AppShell`、`Domain`、`SourceCoordinator`、`Preferences`、`Diagnostics`、`Shared` 以及测试子目录的局部 `AGENTS.md` 增加“学习导向中文注释”的明确要求。
- 这样未来新 session 即使直接进入某个子目录，也会先看到局部规则，而不只依赖根目录 `AGENTS.md`。

### 9. 提升最低支持版本到 macOS 14

- 手工运行时发现“打开设置”使用 AppKit 绕行方案会触发系统报错，明确要求改回 `SettingsLink`。
- 因为 `SettingsLink` 需要 `macOS 14+`，所以当前工程最低支持版本从 `macOS 13` 提升到了 `macOS 14`。
- 相关决策已记录到 ADR：[提升最低 macOS 版本到 14](../adrs/2026-03-30-bump-minimum-macos-to-14.md)。

### 10. Phase 0 严格收口：固定 loopback 回调配置入代码

- 把文档里约定的 `http://127.0.0.1:23388/oauth/callback` 收敛成共享配置类型，而不再只是文字约定。
- 新增 `OAuthLoopbackConfiguration.feishuDefault`，为后续 `BYO Feishu App` 的本地 OAuth loopback server 预留统一来源。
- 增加单元测试锁定 host、port、path 和完整 callback URL，至此 `Phase 0` 在原始定义里的“固定 loopback 端口”也已落地。
